序列埠連線

USB 序列埠連線

1
確認 USB‑Serial 介面卡已插入電腦，驅動程式已安裝，裝置管理員中可看到對應 COM Port。
2
頂部列 Channel 選擇 Comport（預設），Protocol 選擇 UART 或 CAN。
3
（選填）調整 Baud、Data、Stop、Parity。CAN 模式若設為 9600 系統自動升至 460800。
4
點擊「開啟連線」，瀏覽器彈出 COM Port 選擇視窗，選取對應介面卡後點擊「連線」：
file:/// 想要連線到序列埠
VerveBuds 300
serial port (COM10)
USB JTAG/serial debug unit (COM11)
STM32 Virtual ComPort (COM3)
連線 取消

▲ USB 序列埠選擇視窗示意

5
狀態燈變綠，狀態記錄顯示 serialPort open success（UART）或 Set comport to Tool_R OK!（CAN）。
⚠ 開啟失敗常見原因
  • 序列埠被其他程式佔用（如 Endex WinForms 工具）
  • 裝置未連線或驅動程式未安裝
  • 在同一頁面重複開啟（請先按「關閉」再重試）
ℹ SIG 協議比對
連線後工具自動檢查 SIG 末字元奇偶，判斷應使用 UART 或 CAN；若與目前選擇不符會在狀態記錄提示。

BLE 連線

1
頂部列 Channel 選擇 BLE，Baud / Data / Stop / Parity 等序列埠專用設定會自動隱藏。
2
點擊「開啟連線」，瀏覽器彈出 BLE 裝置選擇視窗，只會列出名稱為 ESP_GATTS_DEMO 的裝置（即 ESP32 BLE 橋接器）。
3
選取裝置後，系統可能跳出配對確認視窗，點擊「配對」完成配對：
file:/// 想要配對
▶ESP_GATTS_DEMO
? 正在掃描... 配對 取消

▲ BLE 裝置配對視窗示意

4
配對完成後，工具自動偵測支援的 BLE UART Service/Characteristic（依序嘗試 ESP_GATTS → NUS → HM10 → SPP）。
5
狀態燈變綠，頂部列顯示已連線的 BLE 裝置資訊。
ℹ 找不到裝置
確認 ESP32 BLE 橋接器已開機並正在廣播名稱 ESP_GATTS_DEMO；BLE 連線僅支援 Chrome / Edge（需具備 Web Bluetooth API）。

設定檔載入（INI）

工具啟動時自動 fetch 同目錄的 page.ini 與 parameter.ini，並快取至 localStorage。

自動載入流程

  1. 頁面載入 → fetch page.ini 與 parameter.ini
  2. 成功 → 解析並更新 localStorage，顯示「自動載入成功」
  3. 失敗 → 從 localStorage 讀取舊快取；若無快取則顯示「請手動選擇 INI 檔案」

手動載入

點擊「手動選 page.ini」或「手動選 parameter.ini」從本機選擇對應檔案。

INI 檔案格式

page.ini — 頁籤定義

[Section0]
Key_pagename=SIG Para
Key_pagenum=00

[Section1]
Key_pagename=Motor Para
Key_pagenum=01

parameter.ini — 參數定義

[Section0]
Key_name=SIG0
Key_address=0000
Key_unit=
Key_upperbound=65535
Key_lowerbound=0
Key_tooltip=bit0 Description
Key_writeable=False
Key_hidden=True
Key_setbit=False
Key_production=False
欄位型別說明
Key_name字串參數顯示名稱
Key_address4位元十六進位裝置記憶體位址（如 0000）
Key_unit字串單位標籤（可空白）
Key_upperbound整數 0–65535允許輸入上限
Key_lowerbound整數 0–65535允許輸入下限
Key_tooltip字串SetBit 各 bit 描述，格式 bit0 說明\nbit1 說明
Key_writeableTrue/FalseFalse＝唯讀，不寫入裝置
Key_hiddenTrue/FalseTrue＝使用者模式下隱藏
Key_setbitTrue/FalseTrue＝使用位元設定視窗輸入
Key_productionTrue/FalseTrue＝生產模式下才寫入

讀取裝置參數

1
確認序列埠已連線（狀態燈綠色）。
2
點擊側邊欄「讀取參數」。
3
工具逐一對每個參數位址發送讀取命令，進度條即時更新。
4
完成後顯示「讀取完成」，各頁籤更新為裝置實際值。
ℹ 連續失敗保護
連續失敗 10 個位址後自動停止，並在狀態記錄列出失敗位址清單。

通訊格式

UARTCAN
讀取命令3A 2B 03 04 AH AL 00 00 CRC 0D 0AID 0x03002050，Data [01 03 AH AL 00 00 00 00]
讀取回應相同格式，Data 填入數值ID 0x01005020，Data [01 03 AH AL DH DL 00 00]

寫入裝置參數

1
在各頁籤中修改參數數值（直接輸入或使用 SetBit 視窗）。
2
點擊側邊欄「寫入參數」。
3
工具逐一寫入標記為 Writeable=True 且位址 > 0x0020 的參數。
4
全部寫入後自動執行 SAVE；CAN 模式額外執行 RELOAD。
5
狀態記錄顯示「寫入完成」。
⚠ 注意事項
  • 位址 ≤ 0x0020 的參數視為唯讀，自動跳過。
  • 標記 production=True 的參數在非生產模式下不寫入。
  • 寫入後若裝置重開機前未成功 SAVE，數值會還原。

比對差異

最多可比對三組資料：import1（第一次匯入）、import2（勾選「顯示比較參數」後匯入）、read（讀取裝置）。

1
準備至少兩組資料（任意組合的 import1 / import2 / read）。
2
點擊「⇄ 比對差異」開啟視窗。
3
從「來源 A」與「來源 B」下拉選單選擇要比對的兩組資料。
4
使用篩選按鈕縮小顯示：「全部」「僅差異」「僅相同」「缺少參數」。
5
（選填）點擊「匯出差異報告 (.txt)」下載文字報告。

比對表格欄位：位址 ｜ 名稱 ｜ 來源 A 值 ｜ 來源 B 值 ｜ 差值 (Δ) ｜ 狀態（✗差異 ✓相同 ⚠缺少）

位元設定 (SetBit)

當參數定義為 Key_setbit=True 時，數值欄顯示「位元設定」按鈕。

1
點擊「位元設定」按鈕開啟視窗。
2
視窗顯示 16 個核取方塊（bit 15 在左，bit 0 在右），每個 bit 對應 Key_tooltip 的說明文字。
3
勾選 / 取消對應 bit，下方即時顯示合成的 HEX 值與十進位值。
4
點擊「確定」套用，「取消」放棄。

參數配置編輯器

無需手動編輯 INI 檔，直接在視覺化表格中修改參數定義。

1
點擊側邊欄「參數配置」按鈕。
2
表格呈現所有參數定義，點擊儲存格直接編輯。錯誤格式以紅色底色標示。
3
「新增行」在末尾加入一筆空白參數；「刪除選取行」刪除選取的列。
4
點擊「確定 & 儲存」— 驗證通過後下載更新的 parameter.ini 並立即套用至目前工作階段。
💡 提示
下載的 parameter.ini 應放回工具同目錄，下次開啟時自動載入新定義。

韌體更新（Burn）

⛔ 重要警告
燒錄過程中請勿斷開序列埠或關閉瀏覽器，否則可能造成裝置韌體損毀，需重新燒錄才能恢復。

載入 HEX 檔

1
點擊「📂 載入 HEX 檔」，選擇 .hex 或 .txt 格式的韌體檔案。
2
工具解析第一行取得 SIG 與版本號，格式：@@<SIG8字元>v<VV>.<vv>。
3
成功後顯示檔名、新版本號、行數、位元組數；「確認 Firmware 狀態」與「開始燒錄」按鈕啟用。
ℹ HEX 檔格式說明
工具讀取所有行，遇 @$ 開頭行則截止（含該行）。UART 燒錄只處理 : 或 @ 開頭的行；CAN 燒錄傳送整個檔案文字。

選擇燒錄方式（標準 / BLE-CAN）

「韌體更新」區塊的「方式」下拉選單可選擇燒錄傳輸方式：

選項說明
標準（預設）依目前連線協議，以 UART 逐行或 CAN 逐段方式燒錄。
BLE-CAN透過 BLE 橋接器（ESP32 ble_bridge）以「分塊」方式傳送韌體資料，每塊封裝為 [0xF1, len, 資料]，傳送後等待裝置 ACK 才送下一塊。

選擇「BLE-CAN」後，會顯示「封包」下拉選單，可設定每個分塊的大小：

封包大小說明
124 bytes（預設）較保守，BLE 連線品質不穩時建議使用。
248 bytes分塊數較少、燒錄較快，但需 BLE 連線穩定。
💡 使用建議
若燒錄過程中持續出現逾時或重試，可改選 124 bytes 後重新燒錄。

讀取 DRV FW & 狀態

點擊「🔍 讀取 DRV FW & 狀態」，向裝置查詢目前執行模式與韌體版本（無燒錄動作）。

狀態記錄顯示說明
✅ Bootloader（可燒錄）裝置在 Bootloader 模式，可直接執行燒錄
✅ App（正常運行中）裝置在 App 模式，燒錄前工具會自動跳轉至 Bootloader
❌ 無回應 / SIG 不符逾時或 SIG 識別碼不匹配，需確認協議與連線

執行燒錄

1
確認序列埠已連線且已載入正確 HEX 檔。
2
點擊「⚡ 開始燒錄」。
3
工具自動執行以下流程，可在狀態記錄觀察進度：
  1. 解析 SIG → 查詢 Firmware 狀態
  2. 若在 App → Jump to BootLoader → 再次確認進入 Bootloader
  3. 逐行（UART）或分段（CAN）傳送韌體資料，進度條更新
  4. 燒錄完成 → Jump to App → 確認進入 App 模式
  5. 顯示 ✅ Burn Finished!
ℹ 重試機制
  • UART 單行 ACK 錯誤最多重試 10 次；逾時或整體 CRC 錯誤時無限重試（每次等待 3 秒）。
  • CAN ACK 失敗最多整體重試 5 次，超過後放棄並報錯。
  • BLE-CAN 單一分塊 NAK 最多重試 3 次，仍失敗則中止燒錄；逾時則整體燒錄重新開始（等待 3 秒後從頭傳送）。

停止燒錄

燒錄進行中，按鈕變為「⏹ 停止燒錄」（灰色）。

1
點擊「⏹ 停止燒錄」，按鈕顯示「停止中...」並暫時 disabled。
2
目前行（UART）或分段（CAN）完成後，工具在安全點偵測取消旗標，立即中止並顯示「⏹ 燒錄已終止」。
3
按鈕恢復為「⚡ 開始燒錄」，可重新執行。
⚠ 注意
中途停止後裝置可能停留在 Bootloader 或韌體不完整狀態，需重新執行完整燒錄流程。

  • 診斷讀寫失敗或 CRC 錯誤，確認封包內容是否正確。
