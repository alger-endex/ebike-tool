概述 — 兩個檔案的分工

Endex eBike Tool 的參數顯示介面由兩個 INI 設定檔共同決定，兩者各司其職：

檔案職責影響範圍
page.ini定義 UI 頁籤列表（Tab Bar）頁籤數量、名稱、對應的位址前綴
parameter.ini定義所有硬體暫存器參數每個參數的位址、名稱、單位、上下限、旗標

兩個檔案之間沒有任何明確交叉引用欄位，參數歸屬於哪個頁籤，完全由位址前兩碼與頁籤編號的隱式對應關係決定。

ℹ 設計優點
兩個檔案獨立修改互不干擾：新增頁籤只需修改 page.ini；新增或調整參數只需修改 parameter.ini，只要位址前兩碼吻合即自動歸類到對應頁籤。

page.ini 格式規格

語法結構

標準 INI 格式，每個 [SectionN]（N 從 0 起連續遞增）定義一個頁籤。

欄位說明

欄位鍵名型別說明範例
Key_pagename字串頁籤按鈕顯示文字Motor Para
Key_pagenum2 位十六進位字串頁籤對應的位址前綴，也作為位址範圍篩選依據01
⚠ 注意
  • Key_pagenum 必須恰好 2 位十六進位字元（00–FF），不可為 1 位或 3 位以上。
  • [Section0] 固定對應 SIG 頁籤（pagenum 00），在使用者模式下會隱藏。
  • Section 編號必須從 0 連續遞增，不可跳號；解析器遇到第一個缺少的 SectionN 即停止。

完整範例

[Section0]
Key_pagename = SIG Para
Key_pagenum= 00

[Section1]
Key_pagename = Motor Para
Key_pagenum= 01

[Section2]
Key_pagename = Bike Para
Key_pagenum= 02

[Section3]
Key_pagename = Assist Para
Key_pagenum= 03

[Section4]
Key_pagename = Pedal Sensor
Key_pagenum= 04

[Section5]
Key_pagename = Throttle Para
Key_pagenum= 05

parameter.ini 格式規格

語法結構

標準 INI 格式，每個 [SectionN]（N 從 0 起連續遞增）定義一個參數（HRItem）。

欄位說明

欄位鍵名型別說明
Key_name字串參數顯示名稱，出現在 UI 每列的左側
Key_address4 位十六進位字串裝置記憶體位址，前兩碼決定所屬頁籤（如 0101 → 頁籤 01）
Key_unit字串（可空白）顯示於數值右側的單位標籤（如 0.1V、0.01A）
Key_upperbound整數 0–65535UI 輸入上限；超出時以紅底提示並 clamp 至此值
Key_lowerbound整數 0–65535UI 輸入下限
Key_tooltip字串（\n 換行）滑鼠懸停提示；若 Key_setbit=True，格式必須為 bitN 說明 以供 SetBit 視窗解析
Key_writeableTrue / FalseFalse＝唯讀（顯示為 <span>，不寫入裝置）
Key_hiddenTrue / FalseTrue＝使用者模式下隱藏（工程模式可見）
Key_setbitTrue / FalseTrue＝使用 SetBit 視窗（位元勾選器）取代數字輸入
Key_productionTrue / FalseTrue＝生產模式下才寫入（一般流程跳過）
ℹ Key_address 特別說明
  • 位址範圍 0x0000–0x0020 被視為系統保留（SIG、SN），寫入流程自動跳過。
  • 位址 0x0090–0x0092（Fault Story、Warning Story 等）在頁籤 00（SIG Para）顯示，工程模式才可見。
  • 位址需為有效的 16-bit 無號整數（0000–FFFF），建議統一大寫。

完整範例

[Section15]
Key_name=Rs
Key_address=0101
Key_unit=mohm
Key_upperbound=65535
Key_lowerbound=0
Key_tooltip=
Key_writeable=True
Key_hidden=False
Key_setbit=False
Key_production=False

[Section31]
Key_name=Setting Parameter
Key_address=0200
Key_unit=digit
Key_upperbound=65535
Key_lowerbound=0
Key_tooltip=bit0 : console debug info                 0:disable 1:enable\nbit1 : speed limit from               0:driver  1:console
Key_writeable=True
Key_hidden=True
Key_setbit=True
Key_production=False

兩者隱式對應規則（Address Prefix Mapping）

對應規則說明

頁籤與參數的歸屬關係以下列規則決定：

規則：將參數位址格式化為 4 位大寫十六進位字串，若其前兩碼與某頁籤的 Key_pagenum 相同，則該參數屬於該頁籤。

等效判斷式（JavaScript）：

addrHex.startsWith(tab.pageNum.toUpperCase())

其中 addrHex 為 item.address.toString(16).toUpperCase().padStart(4, '0')。

現有頁籤位址範圍對照

pageNum頁籤名稱位址範圍（十六進位）範圍（十進位）典型參數
00SIG Para 0x0000–0x00FF0–255 SIG0–SIG3、SN0–SN7、Fault Story、Warning Story
01Motor Para 0x0100–0x01FF256–511 Rs、Ls、Poles、Max Voltage、Spd sensor cnts
02Bike Para 0x0200–0x02FF512–767 Wheel Circumference、Speed Limit、Push Assist、Hardware Parameter
03Assist Para 0x0300–0x03FF768–1023 Torque/Power/Speed Level 1–9、Max Current、Power Reduction
04Pedal Sensor 0x0400–0x04FF1024–1279 Sensor Type、cnts per round、Torque mv to nm ratio
05Throttle Para 0x0500–0x05FF1280–1535 Throttle Enable、Throttle Max Power、zero/max voltage
💡 位址命名慣例
目前的參數位址採用 頁碼 × 0x100 + 偏移 的隱性規則。例如 Motor Para（頁碼 01）的所有參數位址落在 0x01xx，偏移由功能群組決定（如馬達阻抗 01xx、保護電流 011x、車速 012x）。

程式碼對應邏輯

實作位於 js/ui.js 的 generateTabs() 函式：

// 取出屬於此頁籤的所有 HRItem
const items = state.hrList
  .map(function(item, idx) { return { item: item, idx: idx }; })
  .filter(function(o) {
    const addrHex = o.item.address.toString(16).toUpperCase().padStart(4, '0');
    return addrHex.startsWith(tab.pageNum.toUpperCase());  // ← 前兩碼比對
  });

過濾結果再依 engineeringMode 決定是否顯示 hidden=True 的參數。

整體資料流

page.ini parameter.ini │ │ ▼ parsePageIni() ▼ parseParameterIni() state.tabList[] state.hrList[] [ { pageName, pageNum }, … ] [ HRItem{ name, address, data, … }, … ] │ │ └──────────────┬───────────────────────┘ │ ▼ generateTabs() [ui.js] 對每個 tab，過濾 hrList： address 前兩碼 === tab.pageNum │ ┌─────────┴──────────┐ ▼ ▼ Tab Panel 0 Tab Panel 1 … (SIG Para) (Motor Para) 過濾出 addr 00xx 過濾出 addr 01xx │ ▼ buildParamRow() 每列：名稱 ｜ 輸入框 ｜ 單位 ｜ Hex checkbox │ ┌──────────────┴──────────────┐ │ 讀取裝置 │ 匯入 Config ▼ ▼ hrList[i].data = 裝置值 paraList[{ address, data }] (用於差異比對與比較顯示)

新增頁籤或參數的操作說明

新增頁籤

只需在 page.ini 末尾加入新 Section：

[Section6]
Key_pagename = Battery Para
Key_pagenum= 06

接著在 parameter.ini 加入位址落在 0x06xx 範圍的參數，重新載入即可看到新頁籤。

⚠ Section 編號連續性
新頁籤的 Section 編號必須接在現有最大編號後一位，不可跳號。例如現有 Section0–Section5，下一個必須是 [Section6]。

新增參數

在 parameter.ini 末尾加入新 Section，確認：

  1. 位址前兩碼對應到想顯示的頁籤 pageNum。
  2. Section 編號接在現有最大編號後一位（不跳號）。
  3. 位址不重複（重複位址不會報錯，但同位址兩個參數都會顯示，讀寫會發生覆蓋）。
[Section99]
Key_name=Battery Voltage
Key_address=0600
Key_unit=0.1V
Key_upperbound=580
Key_lowerbound=0
Key_tooltip=Battery pack total voltage
Key_writeable=False
Key_hidden=False
Key_setbit=False
Key_production=False

常見錯誤與注意事項

情況後果解決方式
位址前兩碼無對應 pageNum參數不出現在任何頁籤（靜默忽略）確認位址前兩碼，或在 page.ini 補充對應頁籤
Section 編號跳號解析器在缺號處停止，後面的 Section 全部遺失確保編號連續從 0 起
位址重複兩個參數都顯示，寫入時後者覆蓋前者搜尋 parameter.ini 確認無重複 Key_address
Key_setbit=True 但 tooltip 格式錯誤SetBit 視窗顯示無說明的 bit0–bit15確認 tooltip 格式為 bitN 說明\nbitM 說明
Upper/Lower Bound 設定反向所有輸入值均顯示超範圍紅底確認 upperbound > lowerbound
Key_address 少於 4 位且帶前導零前兩碼比對失效，歸類錯誤頁籤位址統一補齊 4 位（如 0101 而非 101）

HRItem 資料結構（程式端）

parameter.ini 每個 Section 解析後在 JavaScript 中形成一個 HRItem 物件，定義於 js/ini.js：

屬性來源欄位型別說明
nameKey_namestring顯示名稱
addressKey_addressnumber（uint16）16 進位解析後的整數位址
data—（runtime）number執行期數值，初始為 0；由讀取或匯入 Config 更新
unitKey_unitstring單位標籤
upperBoundKey_upperboundnumber輸入上限（integer）
lowerBoundKey_lowerboundnumber輸入下限（integer）
tooltipKey_tooltipstring滑鼠懸停說明，\n 顯示為換行
writeableKey_writeableboolean是否渲染為可輸入欄位
hiddenKey_hiddenboolean使用者模式是否隱藏
setBitKey_setbitboolean使用 SetBit 視窗輸入
productionKey_productionboolean僅生產模式寫入
nameEl—（ui.js）HTMLElement名稱欄 DOM 參考（generateTabs 後填入）
valueEl—（ui.js）HTMLElement數值欄 DOM 參考
unitEl—（ui.js）HTMLElement單位欄 DOM 參考
hexChkEl—（ui.js）HTMLInputElement十六進位核取方塊 DOM 參考

序列化回 INI 格式時（serializeParameterIni()），data 與四個 DOM 參考不會寫入，僅寫入定義欄位。

INI 快取機制

工具啟動時依下列優先順序載入 INI：

  1. 從伺服器 fetch page.ini 與 parameter.ini（同目錄）。
  2. 成功 → 解析後存入 localStorage，鍵名：
    • endex_page_ini（頁籤定義原始文字）
    • endex_parameter_ini（參數定義原始文字）
  3. fetch 失敗 → 從 localStorage 讀取舊快取；若無快取則顯示「請手動選擇 INI 檔案」。
操作說明
↻ 重新載入強制 fetch 最新版本並更新快取，適合修改 INI 後立即套用
✕ 清除快取刪除 localStorage 中的兩個鍵，下次開啟時重新 fetch
手動選擇 INI從本機選擇檔案，解析後寫入 localStorage（覆蓋快取）
Config 編輯器「確定 & 儲存」將修改後的 HRItem 序列化，更新快取並下載新 parameter.ini
💡 修改 INI 後的建議流程
  1. 將新版 INI 檔案放回工具同目錄（取代舊檔）。
  2. 在工具側邊欄點擊「↻ 重新載入」，即可套用新定義。
  3. 無伺服器環境（直接開啟 HTML）：使用「手動選擇 INI」從本機選取新版檔案。
